# my-games

スマホ向け1画面完結の **2D 4対4コマンドバトル** の最小実装です。

## 1) このゲームの最小仕様まとめ
- 1バトル画面のみ（育成/ガチャ/装備/ストーリー/メニューなし）。
- 味方4体 vs 敵4体の固定バトル。
- 行動順は素早さ順（生存者で毎ラウンド再計算）。
- 味方ターンは3スキルから選択、必要時は対象をタップ指定。
- スキル2/3はクールタイムあり（行動開始時にCTを1減少）。
- 敵ターンは簡易AIが自動選択。
- バフ/デバフ/状態異常は「攻撃UP / 防御DOWN / 気絶 / 継続回復」のみ。
- 敵全滅で勝利、味方全滅で敗北。

## 2) 画面構成
- 左：味方4体（名前・HP・状態表示・行動中ハイライト）。
- 右：敵4体（同上）。
- 中央：ターン情報、状態表示、バトルログ。
- 下：現在行動中の味方のスキル1〜3ボタン。

## 3) データ構造の方針
- `Character`: 基本ステータス、生存状態、3スキル、CT、状態一覧。
- `Skill`: 対象タイプ、CT、説明、効果配列（ダメージ/回復/状態付与）。
- `State`: 味方/敵リスト、行動順、現在アクター、選択中スキル、フェーズ。
- スキル効果は `effects` 配列で処理し、巨大if文の増殖を抑制。

## 4) 戦闘の流れ
1. 開始時に8体を生成し、行動順を初期化。
2. ターン開始時にCT減少、継続回復、気絶判定。
3. 味方ターン：スキル選択 → 必要なら対象選択 → 実行。
4. 敵ターン：AIが使用可能スキルと対象を決定して実行。
5. 行動後に状態ターンを消費し、勝敗判定。
6. 勝敗未決なら次アクターへ。

## 5) 実装ステップ
1. 4対4の表示と行動ハイライト。
2. スキルボタン/CT表示/対象選択UI。
3. ダメージ・回復・状態異常の共通処理。
4. 敵AI（大技優先、低HP狙い、回復判断）。
5. 勝敗表示とリスタート。
6. 画像スプライト（添付画像8体）表示対応。

## 6) 初期キャラ4人＋敵4体の仮スキル案
### 味方
1. **剣士ライル（攻撃役）**
   - S1 スラッシュ：小ダメージ
   - S2 パワーストライク：中ダメージ（CT2）
   - S3 ブレイブバースト：大ダメージ（CT4）
2. **魔法使いミナ（回復役）**
   - S1 ワンドヒット：小ダメージ＋最低HP味方小回復
   - S2 ヒール：味方単体回復（CT2）
   - S3 グループヒール：味方全体回復（CT5）
3. **レンジャーノア（支援役）**
   - S1 アロウショット：小ダメージ＋自分攻撃UP
   - S2 バトルクライ：味方単体攻撃UP（CT3）
   - S3 ラリーソング：味方全体攻撃UP（CT5）
4. **神官セレス（妨害役）**
   - S1 ヘックスジャブ：小ダメージ＋防御DOWN
   - S2 スタンボム：敵単体気絶（CT3）
   - S3 ブレイクストーム：敵全体防御DOWN（CT5）

### 敵
- スライム（攻撃寄り）
- ドラゴン（火力役）
- ゴースト（回復寄り）
- マッシュ（妨害役）

それぞれ同コンセプトで「S1常時 / S2中効果CT / S3大効果CT」を実装。

## 画像について（添付8体）
`app.js` は `assets/party-and-monsters.png` を **2列×4行のスプライトシート** として参照します。

- 左列: 味方4体（上から 剣士 / 魔法使い / レンジャー / 神官）
- 右列: 敵4体（上から スライム / ドラゴン / ゴースト / マッシュ）

添付画像をこのパスに保存すると、そのままゲーム内で8体に適用されます。

## 実行方法
`index.html` をブラウザで開くだけで動作します。


## 操作改善（今回の修正）
- スキル選択後に **即発動しない** 仕様へ変更。
- 単体スキルは対象を選んだあと「発動」ボタンで確定。
- 全体/自分対象スキルも確認ステップを挟むため、誤タップを取り消し可能。
- 「戻る」でスキルを選び直し可能。
- バトルログは最大80行に制限し、古いログを自動破棄。
- ユニットに軽いアニメーション（攻撃/被弾/回復）を追加。

> 補足: `assets/party-and-monsters.png` が存在しない場合でも、カード右下の絵文字で識別できるフォールバック表示を入れています。
